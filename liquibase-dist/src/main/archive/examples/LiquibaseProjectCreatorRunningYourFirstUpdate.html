Liquibase Project Creator: Running Your First Update

A Liquibase changelog has been created to house all your Liquibase changesets. Changesets are equivalent to SQL statements that will update your database.

The main Liquibase command you will use is update, which applies any changes in your changelog to your database.

Step 1: View your changelog file

Open the file ~/LB_WORKSPACE/<project_name>/sample.changelog.sql in a text editor.

In this file you can see a series of changesets. Each changeset is uniquely identified by the id and author fields. Liquibase uses these fields to track what changes have been run.

Step 2: Run Liquibase UpdateSQL

The updateSQL command does not modify your database.  It will only show you the SQL that will be run.  To run your first command:

  1. Open a command-line or Terminal app
  2. Navigate to: ~/LB_WORKSPACE/<project_name>
  3. Run: liquibase updateSQL

You will see a list of SQL commands that will be run against your database.

Step 3: Run Liquibase Update

When you run the update command, Liquibase will evaluate which changesets have not been run against your database, then run them.

To apply the changes to your database run: liquibase update

You should see a message saying: Update has been successful.

Step 4: Check your database

Using a Database Viewer such as DBeaver, SqlPlus, or MSSQL Management Studio, connect to your database.

You should now see the following tables added to your database:

  • DATABASECHANGELOG
  • DATABASECHANGELOGLOCK
  • LIQUIBASE_PERSON
  • LIQUIBASE_COMPANY

By running the update, your database now matches the desired database state as defined by the changelog script.

The DATABASECHANGELOG and DATABASECHANGELOGLOCK tables are Liquibase tables, the LIQUIBASE_COMPANY and LIQUIBASE_PERSON tables were created by the changelog.

Step 5: Run Liquibase History

Now, run: liquibase history 

This command will list the changesets that have been applied to your database.

 

Step 6: Rollback your SQL changes

(Note: rollback is not supported on MongoDB.)

Now run: liquibase rollbackCount 2 

This command will remove the changes applied by Liquibase.

The rollback will not remove the DATABASECHANGELOG and DATABASECHANGELOGLOCK tables. These tables will remain in the database, but the records for changesets that were rolled back will be removed from the DATABASECHANGELOGLOCK table.

You can verify the rollback by checking your database.
